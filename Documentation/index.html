Introduction to Construct 3

Construct 3 is a powerful and user-friendly game development platform that allows users to create games without needing to write traditional code. With its intuitive drag-and-drop interface and a wide range of built-in features, Construct 3 makes game creation accessible to beginners and efficient for experienced developers. This documentation will provide a basic overview to help you get started. For more detailed guides and tutorials, additional resources and links will be provided throughout this document.

Let's start with a few links to help you understand things more easily and quickly.

First, if you're not very familiar with Construct 3, I recommend reading the official documentation. It explains everything in great detail.

Javascript in Construct

While this game is developed in Construct 3, it does not rely solely on the traditional event system. Instead, many parts of the game are built using Construct 3's official JavaScript API. This API allows developers to control layouts, objects, behaviors, and systems directly through JavaScript code, providing greater flexibility and deeper control over the game's logic. It's important to note that while general JavaScript knowledge is helpful, understanding Construct 3's specific scripting API is essential for working with this project. For more information, please refer to the official guide.

Tavern Survivors

Welcome to Tavern Survivors – Fight, Level Up, and Survive!
Dive into an intense roguelite action game where you battle endless waves of enemies, collect powerful weapons, and unlock unique heroes. Each run is different with random item drops from chests and strategic weapon upgrades.

Choose from 8 unique heroes, each with their own playstyle and passive abilities. Equip up to 4 weapons in your deck, level them up during runs, and synergize with items dropped from treasure chests. Defeat the final boss to claim victory!

Experience fast-paced combat with diverse attack types – from piercing arrows to chain lightning, from spinning axes to soul-draining auras. Collect XP gems, gather gold and silver, open chests for random items, and become the ultimate survivor.

The monsters are coming. Are you ready to survive?

This documentation provides a comprehensive guide to help you understand and modify Tavern Survivors. We will explain the game's core systems, heroes, weapons, items, and how to add new content.

Weapons

The game features 20+ unique weapons that can be equipped in a 4-slot deck. Weapons level up during runs through the level-up system (max level 8).

Weapon Types

  • Projectile: Fires bullets/arrows at enemies (Pistol, Longbow, Sniper, Shotgun)
  • Melee: Slashes nearby enemies (Sword, Swift Blade, Champion Blade)
  • Targeted: Strikes specific enemies (Lightning Staff, Dark Blade)
  • Aura: Constant damage around player (Soul Aura, Frost Trail)
  • Orbital: Rotating projectiles/shields (Guardian Shield, Cyclone, Dragon Flame)
  • Special: Unique mechanics (Fire Staff, Bone Throw, Blood Ritual, Boomerang)

Game Systems

Core Game Loop

  • Lobby: Select hero, equip weapons to deck, spend gold on upgrades
  • Game: Fight waves of enemies, collect XP, level up weapons
  • Level Up: Choose weapon upgrades when XP bar fills
  • Chests: Spend silver (run currency) to open chests for random items
  • Boss Fight: Defeat the boss at level 12 to win and earn 200 gold

Currency System

  • Gold: Persistent currency saved between runs. Used to unlock heroes and buy upgrades in lobby.
  • Silver: Run-only currency dropped by enemies. Used to open treasure chests during runs.

Level Up System

  • Collect XP gems from killed enemies to fill XP bar
  • When bar fills, choose from 3 random weapon upgrade options
  • Weapons can be upgraded up to level 8
  • Each level increases damage, reduces cooldown, or adds special effects

Tome System

Tomes are special pickups that provide permanent run bonuses:

  • Silver Tome: +50 silver
  • Crit Tome: +5% crit chance
  • HP Tome: +25 max HP and heal
  • Damage Tome: +10% damage
  • XP Tome: +10% XP gain

Boss System

  • Boss spawns at level 12
  • All regular enemies are cleared when boss appears
  • No new enemies spawn during boss fight (1v1)
  • Boss HP scales with equipped items (4000-6000 HP)
  • Boss attacks with fireballs from random directions
  • Defeating boss grants victory and 200 gold reward

Enemy Types

  • Enemy 1-5: Basic melee enemies with varying stats
  • Enemy 6: Rare enemy type
  • Enemy 7 (Fire Mage): Ranged attacker that shoots fireballs
  • Enemy 8 (Tank): High HP, high damage, slow

Adding New Content

Adding a New Hero

  1. Open scripts/HeroData.js
  2. Add a new hero object to the HEROES constant following this template: 
    new_hero: {
    id: "new_hero",
    name: "Hero Name",
    description: "Hero description",
    attackType: ATTACK_TYPE.RANGED, // or MELEE
    ability: "Passive ability description",
    color: "#hexcolor",
    animPrefix: "new_hero", // animation prefix in Player sprite
    defaultWeapon: "weapon_id",
    stats: {
        health: 120,
        speed: 300,
        damage: 25,
        attackSpeed: 1.0,
        critChance: 0,
        critMultiplier: 2.0
    },
    passive: "passiveType",
    passiveValue: 0.1,
    unlocked: false,
    unlockCost: 500
}
  3. Add hero animations to Player sprite: new_hero_idle and new_hero_run
  4. Add hero portrait to characterPortraits sprite

Adding a New Weapon

  1. Open scripts/WeaponData.js
  2. Add a new weapon object to the WEAPONS constant: 
    "new_weapon": {
    id: "new_weapon",
    name: "Weapon Name",
    description: "Weapon description",
    animationName: "New Weapon", // animation in Effects sprite
    type: "universal",
    weaponType: "projectile", // projectile, melee, aura, targeted, etc.
    baseDamage: 50,
    baseCooldown: 1.0,
    baseRange: 200,
    damagePerLevel: 5,
    cooldownReduction: 0.1,
    effectDuration: 0.3,
    upgrades: [
        { damage: 5, cooldown: 0, desc: "+5 Damage" },
        { damage: 0, cooldown: 0.1, desc: "-0.1s Cooldown" },
        // ... up to 5 upgrades
    ]
}
  3. Add weapon animation to Effects sprite
  4. Add weapon icon to Weapons sprite
  5. Implement weapon logic in scripts/WeaponSystem.js if using new mechanics

Adding a New Item

  1. Open scripts/ItemSystem.js
  2. Add a new item to the ITEMS constant: 
    new_item: {
    id: "new_item",
    name: "Item Name",
    description: "Item effect description",
    effect: "effectType", // existing effect type or new
    value: 10, // effect value per stack
    rarity: "rare" // common, uncommon, rare, epic, legendary
}
  3. Add item animation to Items sprite matching the id
  4. If using a new effect type, implement the logic in applyItemEffects()

Getting Started and Common Questions

Project Structure

  • scripts/main.js - Main entry point, game initialization
  • scripts/GameState.js - Global game state and variables
  • scripts/GameConfig.js - Configuration constants
  • scripts/HeroData.js - Hero definitions
  • scripts/WeaponData.js - Weapon definitions
  • scripts/ItemSystem.js - Item definitions and effects
  • scripts/SaveManager.js - LocalStorage save/load
  • scripts/PlayerController.js - Player movement and damage
  • scripts/EnemyManager.js - Enemy spawning and behavior
  • scripts/WeaponSystem.js - Weapon firing and effects
  • scripts/LevelUpManager.js - Level up UI and upgrades
  • scripts/BossManager.js - Boss spawning and behavior
  • scripts/UIManager.js - UI updates and displays

Common Operations

  • Add gold to player:
    SaveManager.addGold(100)

  • Unlock a hero:
    SaveManager.setHeroUnlocked("vampire", true)

  • Trigger level up:
    XPSystem.levelUp()

  • Spawn an enemy:
    EnemyManager.spawnEnemy1(runtime, x, y)

  • Add an item to inventory:
    ItemSystem.addItem("power_core")

Testing & Debugging

  • Test level up (press L key): Instantly triggers level up screen

  • Test victory (press U key): Instantly triggers victory

  • Reset game progress:
    Open browser console: localStorage.clear(); then refresh

  • Check current state:
    console.log(GameState.state)

Configuration

Key settings in scripts/GameConfig.js:

  • BOSS_SPAWN_LEVEL: Level when boss appears (default: 12)
  • BOSS_HEALTH: Base boss HP (scales with items)
  • ENEMY_SPAWN_RATE: How often enemies spawn
  • XP_PER_LEVEL: XP required to level up
  • CHEST_BASE_COST: Initial silver cost for chests
